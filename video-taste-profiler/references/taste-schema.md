# Taste Schema

Every taste lives in:

```text
taste/<taste-slug>/
  manifest.json
  taste.md
  source-log.md
  prompt-vocabulary.md
```

Use lowercase hyphenated slugs. Do not use a creator's legal name as the slug unless the user explicitly wants that and it is appropriate. Prefer craft-based names such as `warm-rainy-urban-memory`, `deadpan-tabletop-comedy`, or `handdrawn-depth-folk-fantasy`.

## Abstraction Rule

The schema is for a reusable director taste, not a literal source breakdown. Use `source-log.md` for concrete source facts such as filenames, frame notes, transcripts, props, character names, and one-off plot events. Use `taste.md`, `manifest.json.style_vectors`, `manifest.json.analysis_dimensions`, and `prompt-vocabulary.md` for the abstract grammar that can guide new original work.

Bad taste writing: "the video shows a gray cat, a yellow gift box, a pool, and a black end card."

Good taste writing: "a familiar domestic setup is interrupted by a tactile object reveal, expands briefly into fantasy space, then returns to a reaction beat that releases the tension."

Each director dimension should move from evidence to pattern to reusable rule. If a detail cannot be generalized beyond the source, keep it in the evidence log instead of making it a production lock.

## Generalization Gate

Every taste should prove that it learned the source's craft rather than preserving the source's content:

- **Source inventory** belongs in `source-log.md`: concrete characters, props, locations, scenes, captions, dialogue, logos, timestamps, and one-off plot events.
- **Craft function** belongs in `taste.md`: what those details do for attention, emotion, comedy, suspense, rhythm, point of view, space, or platform retention.
- **Substitution rule** belongs in `taste.md` and `prompt-vocabulary.md`: what future projects must invent anew while preserving the craft function.
- **Copy-risk boundary** belongs in `usage_constraints` and negative vocabulary: what must not transfer into generated stories, assets, storyboards, prompts, or renders.

The profile should be specific enough that a director can use it, but abstract enough that a different cast, world, conflict, prop system, shot sequence, and ending image can carry the same taste.

## manifest.json

Required top-level fields stay backward compatible with older taste directories:

```json
{
  "version": 1,
  "slug": "taste-slug",
  "title": "Taste Display Name",
  "source_type": "creator|account|single-video|local-video|mixed|original",
  "source_refs": [],
  "created_at": "ISO-8601 timestamp",
  "updated_at": "ISO-8601 timestamp",
  "aspect_ratios": ["9:16"],
  "duration_range_sec": {"min": 5, "max": 90},
  "confidence": "low|medium|high",
  "style_vectors": {
    "narrative": [],
    "camera": [],
    "editing": [],
    "color_light": [],
    "sound": [],
    "performance": [],
    "platform": []
  },
  "analysis_dimensions": {
    "narrative_structure": [],
    "scene_world": [],
    "mise_en_scene": [],
    "character_performance": [],
    "shot_size": [],
    "camera_angle_position": [],
    "camera_movement": [],
    "composition_visual_design": [],
    "optics_texture": [],
    "lighting": [],
    "color_art_direction": [],
    "editing_rhythm": [],
    "transitions_graphic_links": [],
    "sound_music": [],
    "platform_ai_production": []
  },
  "legal_notes": {
    "public_sources_only": true,
    "no_identity_clone": true,
    "no_shot_for_shot_copy": true,
    "no_specific_plot_copy": true,
    "no_protected_design_clone": true,
    "no_voice_or_music_clone": true,
    "source_inventory_in_source_log_only": true
  },
  "usage_constraints": []
}
```

`analysis_dimensions` is recommended for new profiles and generated by `scripts/new_taste.py`, but older profiles without it remain valid. Keep `style_vectors` as the compact downstream summary; use `analysis_dimensions` for the fuller craft map.
